## Архитектура пайплайна (5-7 строк)

1. `domains.txt` — 50 ICP-matched доменов (B2B SaaS/digital, Россия+СНГ), отобраны вручную.
2. `scraper.py` читает домены, делает GET-запрос к каждому сайту, извлекает title, meta description и первый абзац.
3. Персонализация строится только из фактов сайта: meta > first paragraph > title. Ноль домыслов.
4. Email берётся из mailto-ссылок на странице, иначе — шаблон `info@domain`.
5. Результат — `outreach.csv` с 4 колонками: company, website, email, personalization.
6. Sleep 1 сек между запросами — уважаем серверы, не попадаем в rate limit.
7. Три письма с одной переменной `{{personalization}}` — минимальная персонализация, максимальная воспроизводимость.

## Как масштабировать до 1000 компаний

**1. Async scraping (`aiohttp` + `asyncio`)**
Заменить синхронный `requests` на асинхронный клиент. 50 доменов → ~50 секунд сейчас, 1000 доменов → ~15-20 минут при concurrency=20 вместо 17+ минут.

**2. SQLite-кэш результатов**
Сохранять scraping-результаты по домену в локальную БД. При повторном запуске — пропускать уже обработанные домены. Критично при обогащении большой базы итерациями.

**3. MX-верификация перед отправкой**
Проверять MX-записи домена через `dnspython` — если домен не принимает почту, email-guess не имеет смысла. Убирает ~10-15% невалидных адресов без API.
